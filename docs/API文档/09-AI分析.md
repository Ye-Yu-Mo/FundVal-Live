# AI 分析 API

## 1. 获取 AI 配置

```
GET /api/ai/config/
🔒
```

**响应**:
```json
{
    "api_endpoint": "https://api.openai.com",
    "api_key": "****",
    "model_name": "gpt-4o-mini"
}
```

> `api_key` 脱敏返回 `****`。

---

## 2. 更新 AI 配置

```
PUT /api/ai/config/
🔒
```

**请求体**:
```json
{
    "api_endpoint": "https://api.openai.com",
    "api_key": "sk-xxx",
    "model_name": "gpt-4o-mini"
}
```

---

## 3. 提示词模板列表

```
GET /api/ai/templates/
🔒
```

**查询参数**:

| 参数 | 类型 | 说明 |
|---|---|---|
| `context_type` | string | `fund`(基金分析) / `position`(持仓分析) |

首次访问时自动创建两个默认模板：「基金趋势分析」和「持仓健康度分析」。

**响应**:
```json
[
    {
        "id": 1,
        "name": "基金趋势分析",
        "context_type": "fund",
        "system_prompt": "你是一位专业的基金分析师...",
        "user_prompt": "请分析以下基金的近期表现：\n\n基金代码：{{fund_code}}...",
        "is_default": true,
        "created_at": "2026-01-01T00:00:00Z",
        "updated_at": "2026-01-01T00:00:00Z"
    }
]
```

模板中使用 `{{key}}` 占位符，分析时替换为实际数据。

## 4. 创建模板

```
POST /api/ai/templates/
🔒
```

**请求体**:
```json
{
    "name": "自定义分析",
    "context_type": "fund",
    "system_prompt": "...",
    "user_prompt": "...",
    "is_default": false
}
```

---

```

🔒
```

## 7. AI 分析

```
POST /api/ai/analyze/
🔒
```

**请求体**:
```json
{
    "template_id": 1,
    "context_type": "fund",
    "context_data": {
        "fund_code": "000001",
        "fund_name": "华夏成长混合",
        "fund_type": "混合型",
        "latest_nav": "1.4070",
        "estimate_growth": "0.32"
    }
}
```

模板中的 `{{key}}` 占位符会被 `context_data` 中对应的值替换。

**响应**:
```json
{
    "result": "根据数据分析，华夏成长混合近期表现..."
}
```

> 超时: 60s。需要先配置 AI 接口。

---

## 8. AI 投资报告预览

```
POST /api/ai/report-preview/
🔒
```

**请求体**:
```json
{
    "period": "monthly"
}
```

| period | 说明 |
|---|---|
| `weekly` | 本周报告 |
| `monthly` | 本月报告 |
| `yearly` | 本年报告 |

**响应**:
```json
{
    "result": "# 投资月报\n\n## 账户总览\n..."
}
```

> 超时: 180s。报告自动包含账户总览、持仓明细、期间盈亏、表现最佳/最差基金。

---

## 定时报告

Celery Beat 定时生成投资报告并推送到用户通知渠道：

| 任务 | 频率 | 说明 |
|---|---|---|
| `generate-weekly-report` | 每周一 9:00 | 周报 |
| `generate-monthly-report` | 每月 1 日 9:00 | 月报 |

> 需在用户偏好中开启 `report_enabled` 并配置 AI 接口和通知渠道。
